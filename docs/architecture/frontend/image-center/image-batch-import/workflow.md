# 影像中心批量导入交互流程

## Overlay

点击影像中心“批量导入”直接打开 Overlay，不再先弹系统文件选择器。

Overlay 包含：

- `新建导入`：选择文件、患者、检查类型、团队归属和左右翻转。
- `导入任务`：分页展示历史批次及选中批次的文件状态。

## 新建导入

1. 用户选择文件，前端按照后端配置校验数量。
2. 需要左右翻转的普通图片顺序预处理；DICOM 等文件保持原样。
3. 创建批次和所有 item。
4. 每次为最多 10 个 item 申请上传会话。
5. 会话窗口内最多并发上传 3 个文件。
6. 每个文件完成分片上传后立即调用单 item complete。
7. complete 返回后，该文件已经进入 AI 队列；前端继续处理下一个文件。
8. 单文件失败会写入后端，不中止其他文件。
9. 浏览器上传结束后自动切换到“导入任务”。

上传过程中不能关闭 Overlay，避免丢失浏览器中的 File 和 multipart 上下文。
上传阶段结束后可以关闭。

## 任务查询

- 批次列表每页 10 条。
- 文件进度每页 20 条。
- 存在非终态批次时，每 2 秒刷新当前批次页和当前文件页。
- 批次终态为 `COMPLETED`、`PARTIAL_FAILED` 或 `FAILED` 后停止主动轮询。
- AI 失败，或上传完成但 Kafka 排队失败的 item 显示“重新入队”。
- 重新入队不重新上传文件，也不创建新的影像记录。

## 失败处理

| 失败阶段 | 行为 |
| --- | --- |
| 浏览器图片转换 | 标记单文件失败，继续准备其他文件 |
| 会话创建 | 当前调用失败，用户保留错误提示 |
| multipart 上传 | 调用 `upload-failed`，继续同一窗口其他文件 |
| complete 校验 | item 标记失败，不进入 AI |
| Kafka 发布 | item 保持已上传和待排队，可重新入队 |
| AI 模型业务错误 | 显示 AI 失败，可人工重新入队 |
| 页面刷新 | 上传中的本地文件无法恢复；已入队任务可在任务 Tab 查看 |

影像中心列表会在浏览器上传阶段结束后刷新；AI 完成状态通过任务 Tab独立更新。
