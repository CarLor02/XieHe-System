# 测量项与关键点双向绑定规则

## 目的

测量项与解剖关键点描述的是同一组实际点位。它们虽然分别保存在测量项和检测层中，但在业务上不能成为两套互不关联的数据。

双向绑定是影像标注页的通用业务规则。只要一个测量项能够明确确定其测量点对应的关键点，该测量项就必须建立双向绑定，不得只实现单向写回或维护独立坐标副本。

双向绑定用于保证：

- 移动测量项的绑定点时，对应关键点同步移动。
- 移动、纠正或补充关键点时，已绑定测量项同步更新并重新计算结果。
- 医生调整测量项后，对应关键点的来源变为手动标注。
- 测量项的交互辅助点不会被错误地写成解剖关键点。
- 加载历史数据时，可以从已有测量项补齐缺失的绑定关键点。

当前统一绑定规则注册表已接入正位影像中的 `CA` 和 `L/R`。其他能够确定关键点映射的测量项也应逐步接入同一契约。

Cobb 的端椎在用户填写前无法确定，因此使用动态端椎映射和“同步检测层”流程；端椎确定后，它仍须遵守本规则中的双向同步、数值重算、测量项 ID 稳定和缺失依赖处理原则。

## 适用性判断

测量项满足以下任一条件时，必须建立双向绑定：

- 测量配置固定声明了每个测量点对应的关键点。
- 测量点对应的关键点可以通过端椎、顶椎或其他测量参数唯一确定。
- 测量项由一组确定的关键点自动派生。
- 测量项完成后，可以无歧义地从几何顺序或解剖顺序确定关键点语义。

只有在测量点不代表解剖关键点，或当前业务参数不足以唯一确定映射时，才可以不执行绑定。此类例外必须在对应测量工具的业务文档中说明原因。

## 通用约束

1. 绑定规则使用标准化后的测量类型 ID 匹配，避免显示名称、大小写或编号影响规则命中。
2. 双向同步只更新点位、测量值及绑定状态，不修改测量项的 `id` 和 `type`。
3. 测量项写回关键点时，关键点使用：
   - `source = MANUAL`
   - `confidence = 1`
4. 关键点写回测量项后，必须使用当前测量配置重新计算 `value`。
5. 已建立绑定的测量项标记为 `keypointSynced = true`。
6. 对于已经存在的关键点，历史数据回填不得覆盖其坐标、来源或置信度。
7. 同一映射不得分别散落在测量工具、拖拽组件和关键点工作流中，必须通过统一绑定规则或遵循相同契约的专用适配器实现。
8. 能够确定关键点映射却没有建立双向绑定，应视为功能缺陷，而不是允许长期存在的工具差异。

## 当前已注册的绑定规则

| 测量项 | 测量项点位 | 绑定关键点 | 关键点完整条件 |
| ------ | ---------- | ---------- | -------------- |
| CA     | 2 个点     | `CR`、`CL` | `CR`、`CL` 均存在 |
| L/R    | 4 个解剖锚点和 8 个纵线端点 | `ASIS_L`、`SI_L`、`SI_R`、`ASIS_R` | 四个关键点均存在 |

## CA 绑定

### 点位映射

| CA 测量项点位 | 关键点 |
| ------------- | ------ |
| `points[0]`   | `CR`   |
| `points[1]`   | `CL`   |

### CA 到关键点

- 手动完成 CA 测量时，两个测量点分别写入 `CR` 和 `CL`。
- 拖动 CA 的一个端点时，只更新该端点绑定的关键点。
- 未被拖动的另一个关键点保持原有坐标、来源和置信度。

### 关键点到 CA

- `CR` 或 `CL` 被拖动、纠正或重新标注后，CA 的两个测量点按上表重新组装。
- CA 保持原有测量项 `id`，并根据新点位重新计算角度。
- CA 的更新不依赖测量项是否由 AI 创建，也不依赖其 `id` 前缀，因此手动 CA 和历史 CA 使用相同规则。

## L/R 绑定

### 关键点语义顺序

L/R 的四条纵线始终按影像屏幕中的 X 坐标从左到右排序，并绑定为：

| 水平顺序 | 解剖含义 | 关键点 |
| -------- | -------- | ------ |
| 1        | 左外缘（ASIS） | `ASIS_L` |
| 2        | 左内缘（SI）   | `SI_L`   |
| 3        | 右内缘（SI）   | `SI_R`   |
| 4        | 右外缘（ASIS） | `ASIS_R` |

X 坐标相同时，使用测量项中原始纵线分组序号作为稳定排序条件。

### L/R 点位结构

L/R 测量项共保存 12 个点：

- `points[0..3]`：医生确定的四个解剖锚点。
- `points[4..11]`：四条纵线各自的上、下端点，用于调整纵线长度。

解剖语义由当前水平排序确定，不由锚点在 `points[0..3]` 中的数组位置固定决定。当两条纵线左右交叉后，系统按新的水平顺序重新确定四个关键点的语义。

### L/R 到关键点

- 手动完成 L/R 测量时，系统先对四条纵线进行水平排序，再写入四个绑定关键点。
- 拖动任意解剖锚点或整条纵线时，重新排序并更新四个绑定关键点。
- 拖动纵线的上、下端点只改变显示长度，不写回关键点。
- 因此，纵线长度属于测量项交互数据，不属于解剖关键点数据。

### 关键点到 L/R

- 四个关键点全部存在时，可自动派生一个全局唯一的 L/R 测量项。
- 任意绑定关键点移动后，系统将对应纵线整体平移到新锚点。
- 平移时同时移动该纵线的上、下端点，保留医生调整后的上下距离和纵线长度。
- 如果原测量项不是合法的 12 点结构，则根据四个关键点重新创建默认长度的四条纵线。
- 更新后重新计算 `dL`、`dR` 和 `L/R`。

## 触发时机

### 测量项写回关键点

以下操作会触发测量项到关键点的同步：

- 在正位“手动标注”模式下完成 CA 或 L/R 测量。
- 拖动 CA 的测量端点。
- 拖动 L/R 的解剖锚点或纵线主体。

单独拉长或缩短 L/R 纵线不会触发关键点更新。

### 关键点写回测量项

以下操作会触发关键点到测量项的同步和数值重算：

- 拖动检测层关键点。
- 使用“椎体点位纠正”调整绑定关键点。
- 手动补充绑定关键点。
- 其他会更新当前内存关键点集合的标注操作。

## 缺失点与删除

- 已绑定测量项依赖的关键点缺失后，该测量项不能继续计算，应从当前测量列表中移除。
- CA 缺少 `CR` 或 `CL` 时移除 CA。
- L/R 缺少四个绑定关键点中的任意一个时移除 L/R。
- 该规则适用于 `keypointSynced = true` 的测量项以及当前 AI 测量产生的绑定测量项。
- 删除关键点后的同步只处理当前内存状态，不改变测量项 ID 复用或编号规则。

## 历史数据兼容

旧标注可能只保存了 CA 或 L/R 测量项，没有保存对应关键点。加载这类数据时：

1. 从已有测量项几何中推导绑定关键点。
2. 只补充当前不存在的关键点。
3. 已存在关键点始终优先，不被历史测量项覆盖。
4. 回填关键点标记为手动来源，置信度设为 1。
5. 回填完成后，再按正常的关键点到测量项流程统一重算。

该兼容只修复当前内存中的标注状态，不要求迁移已有测量项 ID，也不修改后端数据结构。

## 接入新测量项

新增或维护测量项时，应先进行适用性判断。能够确定相关关键点的测量项，必须在统一规则注册表中定义：

1. 标准化的测量类型 ID。
2. 完整计算所需的关键点 ID 列表。
3. 测量项点位写回关键点的映射函数。
4. 关键点重新组装测量项点位的函数。
5. 哪些点是解剖点，哪些点只用于绘制或交互。
6. 点位不完整时应移除测量项还是保留旧数据。
7. 对历史标注是否需要执行缺失关键点回填。

对于 Cobb 这类需要运行时参数才能解析关键点的测量项，可以使用专用适配器，但适配器必须提供与统一规则等价的双向转换和缺失依赖处理。

不得在拖拽组件、测量面板或关键点面板中分别硬编码同一套映射。
