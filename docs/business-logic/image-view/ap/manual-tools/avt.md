# 顶椎平移量（AVT）

## 适用范围

- 仅适用于正位 X 光片标注。
- 工具名称为 `AVT`，含义为顶椎平移量（Apical Vertebral Translation）。
- AVT 是选择式关键点绑定测量项，医生选择顶椎后由系统生成，不需要在画布上自由放点。
- 一张影像可以存在多个 AVT，但同一顶椎最多对应一个 AVT。
- AVT 距离使用影像的标准距离换算为毫米；未设置标准距离时，沿用标注页现有的标准距离提示流程。

## 六点结构

AVT 测量项保存 6 个点：

| 点位索引 | 来源 | 作用 |
| --- | --- | --- |
| 0 | 被选顶椎关键点 1 | 顶椎左上角点 |
| 1 | 被选顶椎关键点 2 | 顶椎右上角点 |
| 2 | 被选顶椎关键点 3 | 顶椎左下角点 |
| 3 | 被选顶椎关键点 4 | 顶椎右下角点 |
| 4 | SR | 骶骨参考点 SR |
| 5 | SL | 骶骨参考点 SL |

顶椎中心取 `points[0..3]` 四个角点坐标的平均值。CSVL 参考位置取 `points[4..5]` 的中点。

## 创建规则

1. 当前影像必须同时存在 SR 和 SL。
2. 顶椎必须具有当前关键点 catalog 声明的完整四个角点。
3. 医生点击 AVT 工具后，从可用椎体列表中选择一个顶椎。
4. 已经存在 AVT 的顶椎置灰且不可重复选择；其他完整顶椎仍可继续创建 AVT。
5. 创建结果在左上角列表显示为 `AVT(<顶椎>)`。

AVT 不使用全局序号。实例身份由 measurement `id` 和 `apexVertebra` 共同确定。

## 计算规则

- 顶椎参考 X 坐标为顶椎四角点中心的 X 坐标。
- CSVL 参考 X 坐标为 SR、SL 中点的 X 坐标。
- 像素偏移量为“顶椎中心 X - CSVL 参考 X”。
- 系统使用标准距离换算偏移量的绝对值，再保留原始左右方向符号。
- 顶椎中心位于 CSVL 左侧时结果为负，位于右侧时结果为正。

## 图像显示

- 使用虚线连接被选顶椎的四个角点，并显示顶椎中心。
- 经过顶椎中心绘制垂直引导线。
- 显示 SR、SL 骶骨参考线及其中点，并经过中点绘制 CSVL 垂直引导线。
- AVT 数值同时显示在影像和左上角测量结果列表中。
- 影像中的 AVT 数值标签显示在被选顶椎四角点区域的右侧，并与四角点中心等高；SR、SL 的位置不参与标签定位。
- 多个 AVT 分别以各自顶椎为标签锚点，避免共享 SR、SL 导致标签集中重叠。

## 绑定与调整规则

- AVT 的六个点均与对应关键点双向绑定。
- 拖动 AVT 的顶椎角点时，写回被选顶椎对应角点；拖动骶骨参考点时，写回 SR 或 SL。
- 移动、纠正或重新标注顶椎角点、SR、SL 后，所有依赖这些点的 AVT 保持原 `id` 和 `apexVertebra` 并重新计算。
- 移动未被某条 AVT 绑定的其他椎体，不影响该 AVT。
- 关键点绑定模式下不允许整体拖动 AVT，避免测量项与关键点层分离。

## 缺失点与删除

- SR 或 SL 缺失时，当前影像中的所有绑定 AVT 均无法继续计算，应从当前测量列表移除。
- 某个顶椎关键点不完整时，只移除绑定该顶椎的 AVT，不影响其他顶椎的 AVT。
- 用户从结果列表删除一条 AVT 时，只删除该 measurement，不删除其顶椎、SR 或 SL 关键点。

## 历史数据兼容

- 历史 AVT 可能使用固定 ID `ap-keypoint-avt`，重算时必须保持该 ID 不变。
- 新创建的 AVT 使用包含顶椎标签的稳定 ID，以区分不同顶椎实例。
- 历史两点格式 `[apexCenter, csvlRef]` 继续保留计算、图形和标签定位兼容；标签使用 `apexCenter` 定位，但不能在缺少 `apexVertebra` 时自动推导出新的顶椎绑定。
