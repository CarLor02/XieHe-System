# 文件索引服务 - Windows 部署说明

## 📦 文件清单

```
communication/
├── 启动文件索引服务.bat          # 标准启动脚本（显示命令行窗口）
├── 启动文件索引服务(静默).vbs    # 静默启动脚本（无窗口，后台运行）
├── 停止文件索引服务.bat          # 停止服务脚本
├── config_manager.html           # 配置管理界面
└── file_index_service/           # 服务核心代码
    ├── main.py
    ├── requirements.txt
    └── ...
```

## 🚀 快速开始

### 方式一：直接启动（推荐）

1. **双击运行**：`启动文件索引服务.bat`
2. 脚本会自动：
   - 检测网络 IP 地址（优先 192.168.189.x）
   - 检查 Python 环境（支持 conda/python3/python）
   - 安装依赖包（如缺失）
   - 创建默认配置文件
   - 启动服务
   - 自动打开配置管理界面

3. 看到 "服务启动中" 后，配置管理页面会自动打开

### 方式二：静默启动（无窗口）

1. **双击运行**：`启动文件索引服务(静默).vbs`
2. 服务在后台运行，无命令行窗口
3. 手动打开配置界面：双击 `config_manager.html`

### 停止服务

双击运行：`停止文件索引服务.bat`

---

## ⚙️ 配置说明

### 首次配置

1. 服务启动后，配置管理界面会自动打开
2. 在 "扫描目录" 栏：
   - **方式 1**：点击快捷路径按钮，修改为实际路径
   - **方式 2**：直接输入完整路径（如 `D:\DICOM_Files`）
3. 设置扫描间隔（秒），默认 300 秒
4. 点击 "保存配置"
5. 重启服务使配置生效（双击 `停止文件索引服务.bat` → 再次启动）

### 手动配置（高级）

编辑 `file_index_service\.env` 文件：

```ini
WATCH_PATH=D:\DICOM_Files        # 扫描目录
SCAN_INTERVAL=300                # 扫描间隔（秒）
DB_PATH=./file_index.db          # 数据库路径
LOG_LEVEL=INFO                   # 日志级别
API_KEY=                         # API 密钥（留空禁用鉴权）
HOST=0.0.0.0                     # 服务监听地址
PORT=9000                        # 服务端口
PRIMARY_EXTENSIONS=.dcm,.dicom   # 主影像文件扩展名
```

## 🔧 常见问题

### 1. 启动后显示"未找到 Python 环境"

**解决方法**：
- 安装 Python 3.8+：https://www.python.org/downloads/
- 安装时勾选 "Add Python to PATH"
- 或者配置 conda 环境并命名为 `dp`

### 2. 服务启动后无法访问

**检查步骤**：
1. 确认防火墙允许端口 9000
2. 在浏览器访问：`http://localhost:9000/health`
3. 如果返回 `{"status":"ok"}` 说明服务正常

### 3. 配置管理界面无法连接服务

**解决方法**：
1. 确认服务已启动
2. 在配置界面的 "服务地址" 栏输入正确的 IP:端口
3. 如果是本机访问，使用 `http://127.0.0.1:9000`
4. 如果是局域网访问，使用实际 IP（如 `http://192.168.189.20:9000`）

### 4. 扫描不到文件

**检查清单**：
- [ ] 扫描目录路径正确
- [ ] 目录下有 `.dcm` 或 `.dicom` 文件
- [ ] 文件直接放在配置的目录下（不要放在子文件夹）
- [ ] 点击 "立即扫描" 手动触发一次扫描

### 5. 如何查看日志

日志文件位置：`file_index_service\logs\file_index.log`

## 📝 部署到其他 Windows 机器

1. 复制整个 `communication` 文件夹到目标机器
2. 确保目标机器已安装 Python 3.8+
3. 双击 `启动文件索引服务.bat`
4. 首次启动会自动安装依赖

## 🌐 网络配置

### 局域网访问

如果需要从其他设备访问此服务：

1. 确保 `.env` 文件中 `HOST=0.0.0.0`
2. 在 Windows 防火墙中允许端口 9000：
   ```
   控制面板 → Windows Defender 防火墙 → 高级设置
   → 入站规则 → 新建规则 → 端口 → TCP 9000
   ```
3. 在其他设备浏览器访问：`http://<服务器IP>:9000`

### 修改端口

编辑 `file_index_service\.env`，修改 `PORT=9000` 为其他端口
